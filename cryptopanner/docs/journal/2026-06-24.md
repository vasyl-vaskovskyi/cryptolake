# 2026-06-24 — Dev-stack audit + first runnable §14.e soak

Audited why `make dev-up` / the soak couldn't run, fixed the blocking code gap (TDD), and landed a
process-based single-node soak that runs green end-to-end. Straight to `main`.

## Context / decisions
- **Process-based over docker-compose (for now).** The §14.e *value* is the orchestration +
  pass-criteria assertions, identical across process/container. The chaos `_common.sh` process
  pattern is proven and runnable here; the docker path needs a Dockerfile + two nonexistent mock
  images. Bring-up is isolated in `soak::stack_up` for a later `docker compose` swap. Two nodes is
  the natural docker boundary (hardcoded `/tmp` collector paths collide on one host).
- **Empirical debugging paid off.** Two real bugs in the runner surfaced only by running it:
  the sealer/uploader exit non-zero on empty REST streams (tolerated), and the uploader deletes
  local sealed files on the durability handoff (so the sealed-pair snapshot must precede upload).

## Findings → next increment
- **Mock needs wall-clock event-time rewriting** (+ unique seq IDs) so frames fill consecutive
  real-time minutes — the enabler for sustained-load (`late_frames`<0.1%, full-hour completeness)
  and reliable rotation-overlap sealing. The fixed-event-time fixture is why `rotation recorded:
  no` and why verify is scoped to captured streams.
- **Sealer/uploader should record an empty hour as a gap-manifest** instead of hard-failing (spec
  "explicit gap surfacing"). TDD bugfix.
- **Docker + two-node variant**: Dockerfile (service-dispatch entrypoint), `mock-healthchecks`,
  fold `mock-binance-rest` into the ws mock, compose collector slot-env fix, two independent nodes
  with own S3 prefixes (§3.d). Then re-point `soak::stack_up` at `docker compose`.

## Session 2 — mock wall-clock event-time rewrite (the soak enabler)

- `feat(mock)` **`EventTimeRewriter`** (TDD, 7 tests). Restamps a replayed frame's uppercase
  `E`/`T` to the send instant, leaving everything else byte-identical (lowercase event-type `e`,
  ids `t`, depth pu-chain `U`/`u`/`pu`, quoted prices). Wired into `WsServer` behind an opt-in flag
  (default off, back-compat constructor); `Main` reads `MOCK_REWRITE_EVENT_TIME`.
- **Soak now uses it** (`MOCK_REWRITE_EVENT_TIME=1`, window bumped to 120s) + asserts progression.
  Re-run green: captured **today's date** across **3 distinct real-time minutes** (was the fixture's
  fixed 2024-06-14 16:00), and **11 streams** verify clean — the REST-derived streams
  (openInterest/depthSnapshot/exchangeInfo) now seal+verify because their receive-date matches the
  WS date.

### Context / decisions
- **Why only `E`/`T`, restamped per-frame:** server-event-time bucketing (§8.c) keys on `E` (or `T`
  for trade/aggTrade); restamping those to "now" makes frames fill consecutive wall-clock minutes
  monotonically, so minutes seal *during* the run (not just at shutdown). Sequence fields are
  untouched, so verify's pu-chain continuity still holds — confirmed by 11× ERRORS=0.
- **Rotation still `no`.** The rewrite fixed minute progression, but each WS connection gets an
  *independent* replay, so a rotation shadow's overlap minute diverges from the primary's and
  equivalence rarely passes. Reliable rotation needs **synchronized dual-connection fan-out**
  (§14.c) — the next mock enhancement.

## Session 3 — gap-surfacing fix + synchronized fan-out

- `fix(sealer)` **Empty hour → zero-record manifest, not a hard fail** (§12.k, TDD). `mergeHour`
  threw "no segments directory" when a configured stream captured nothing, so the sealer exited
  non-zero and the uploader then failed on "missing sealed files". Now an empty hour seals a
  zero-record file + manifest. Cascade: the soak gates strictly on sealer+uploader exit 0 — green,
  11 streams verify ERRORS=0.
- `feat(mock)` **Synchronized dual-connection fan-out** (`FanoutBroadcaster`, §14.c, TDD). `WsServer`
  refactored from per-connection independent replay to a single replay loop that builds each
  (rewritten) frame once and fans byte-identical bytes to all clients — so a rotation primary +
  shadow see the same overlap minute and equivalence passes. Soak re-validated green (capture path
  unaffected).

### Context / decisions
- **Scouting redirected the plan.** Before broad mock fault-injection, scouted §12 coverage: the
  failure-mode *behaviors* are already unit-tested (`BinanceWsClientReconnectTest` §12.b,
  `WsConnectionManagerTest` §12.j, …); only named `*IT` end-to-end scenarios are absent. So
  fault-injection was lower-value than assumed → pivoted to the bounded, high-value fan-out.
- **Fan-out is necessary but not sufficient for the soak's rotation gate.** Root cause the soak still
  shows `rotation recorded: no`: the operator trigger is **refused until the connection is older
  than `minOperatorAge` (5min default, hardcoded in `WsConnectionManager.Config.defaults`)**. A
  short soak triggers at ~45s → refused. Closing this needs a >6min run or a configurable
  (dev-lowered) `minOperatorAge`.

## Session 4 — rotation recorded at real-env; the soak hard-gates it

Closed the loop: the make-before-break WS rotation now completes in the soak and is a hard gate.
Three changes, each surfacing the next blocker:

- `feat(config)` **`collector.min_operator_rotation_age` knob** (§5.4, TDD). `minOperatorAge` was
  hardcoded 5min in `WsConnectionManager.Config.defaults`, so a short run's operator trigger was
  refused as too-young. Now configurable (defaults 5min) via a `Config.withMinOperatorAge` factory;
  dev sets `20s`. → trigger accepted.
- `fix(collector)` **Overlap waits for the primary sibling to seal** (race, TDD regression). With
  the trigger accepted, the rotation reached the merge step and failed reading the *primary's*
  `minute-NN.jsonl.zst`: `scanSealedShadowMinute` returned a minute as soon as the `.shadow` segment
  sealed, but the primary sibling seals on independent timing. Now a minute is overlap-ready only
  when every shadow segment has a sealed primary sibling. → rotation completes.
- `test(soak)` **Early trigger + rotation hard gate.** Trigger at ~30s (just past the 20s dev
  min-age), leaving ~150s for the overlap minute to seal + cut over. Result:
  `rotation_started → rotation_verify_passed → rotation_promoted`, `COMPLETED (PASS)`,
  `rotation recorded: yes`. The soak now fails if no rotation is recorded (§8.b.2/§14.e) unless
  `SOAK_REQUIRE_ROTATION=0`.

### Context / decisions
- **Two real bugs surfaced only by driving a live rotation** — the hardcoded min-age and the
  shadow/primary seal race. The race fix matters in production too: any time the shadow seals an
  overlap minute slightly before the primary, the old code would abort the rotation.
- The fan-out (session 3), the config knob, the race fix, and the timing all had to line up; each
  fix moved the rotation one stage further (refused → aborted-no-minute → failed-at-merge →
  COMPLETED).

## Session 5 — per-node independence insurance + verify CLI inspectors

Reconsidered the docker/two-node soak: nodes have **no inter-node orchestration** (the only
cross-node component is the Monitor — observe/restart/alert-correlate — already unit-tested), so a
runtime two-node test mostly proves "no collision", which is structural. Took the cheap path
instead and moved to real missing functionality.

- `test(common)` **Per-node S3-key independence** (§3.d). Explicit unit assertion that two nodes
  capturing identical data write to distinct node-prefixed keys — the property the two-node soak
  would have exercised, at unit cost.
- `feat(verify)` **`manifest` + `gaps` subcommands** (§14.f, TDD). The CLI had only the integrity
  `verify` subcommand. Added two read-only operator inspectors over the S3 manifests, sharing an
  `S3InspectOptions` @Mixin: `manifest` pretty-prints one (symbol,stream) manifest; `gaps` reports
  `sequence_gaps` + missing-minutes for one stream or all `--config` subscriptions with a `GAPS=<n>`
  summary. 4 tests against in-process S3Mock.
