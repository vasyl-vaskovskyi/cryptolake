# 2026-06-24 — Dev-stack audit + first runnable §14.e soak

Audited why `make dev-up` / the soak couldn't run, fixed the blocking code gap (TDD), and landed a
process-based single-node soak that runs green end-to-end. Straight to `main`.

## Context / decisions
- **Process-based over docker-compose (for now).** The §14.e *value* is the orchestration +
  pass-criteria assertions, identical across process/container. The chaos `_common.sh` process
  pattern is proven and runnable here; the docker path needs a Dockerfile + two nonexistent mock
  images. Bring-up is isolated in `soak::stack_up` for a later `docker compose` swap. Two nodes is
  the natural docker boundary (hardcoded `/tmp` collector paths collide on one host).
- **Empirical debugging paid off.** Two real bugs in the runner surfaced only by running it:
  the sealer/uploader exit non-zero on empty REST streams (tolerated), and the uploader deletes
  local sealed files on the durability handoff (so the sealed-pair snapshot must precede upload).

## Findings → next increment
- **Mock needs wall-clock event-time rewriting** (+ unique seq IDs) so frames fill consecutive

  and reliable rotation-overlap sealing. The fixed-event-time fixture is why `rotation recorded:
  no` and why verify is scoped to captured streams.
- **Sealer/uploader should record an empty hour as a gap-manifest** instead of hard-failing (spec
  "explicit gap surfacing"). TDD bugfix.
- **Docker + two-node variant**: Dockerfile (service-dispatch entrypoint), `mock-healthchecks`,
  fold `mock-binance-rest` into the ws mock, compose collector slot-env fix, two independent nodes
  with own S3 prefixes (§3.d). Then re-point `soak::stack_up` at `docker compose`.

## Session 2 — mock wall-clock event-time rewrite (the soak enabler)

- `feat(mock)` **`EventTimeRewriter`** (TDD, 7 tests). Restamps a replayed frame's uppercase
  `E`/`T` to the send instant, leaving everything else byte-identical (lowercase event-type `e`,
  ids `t`, depth pu-chain `U`/`u`/`pu`, quoted prices). Wired into `WsServer` behind an opt-in flag
  (default off, back-compat constructor); `Main` reads `MOCK_REWRITE_EVENT_TIME`.
- **Soak now uses it** (`MOCK_REWRITE_EVENT_TIME=1`, window bumped to 120s) + asserts progression.
  Re-run green: captured **today's date** across **3 distinct real-time minutes** (was the fixture's
  fixed 2024-06-14 16:00), and **11 streams** verify clean — the REST-derived streams
  (openInterest/depthSnapshot/exchangeInfo) now seal+verify because their receive-date matches the
  WS date.

### Context / decisions
- **Why only `E`/`T`, restamped per-frame:** server-event-time bucketing (§8.c) keys on `E` (or `T`
  for trade/aggTrade); restamping those to "now" makes frames fill consecutive wall-clock minutes
  monotonically, so minutes seal *during* the run (not just at shutdown). Sequence fields are
  untouched, so verify's pu-chain continuity still holds — confirmed by 11× ERRORS=0.
- **Rotation still `no`.** The rewrite fixed minute progression, but each WS connection gets an
  *independent* replay, so a rotation shadow's overlap minute diverges from the primary's and
  equivalence rarely passes. Reliable rotation needs **synchronized dual-connection fan-out**
  (§14.c) — the next mock enhancement.
