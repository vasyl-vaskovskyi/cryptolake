## Context / decisions
- **Why a tracked journal at all.** `.remember/` is gitignored (verified: `.remember/.gitignore` excludes the whole directory), so daily logs, WIP buffer, and rolling summaries do not travel between machines. The git log + master spec + walking-skeleton plan were the only artefacts surviving a `git pull` on a fresh laptop. Journal closes that gap. Per-day file, append-only, terse by convention; multiple sessions on the same day go into the same file with `## Session HH:MM` sub-headings.
- **Why the final collapse to just the soak.** Decomposing each of the original 18 failure modes rigorously showed that every one separates into pure-logic (unit) + wiring (integration) coverage. The three "real-env keepers" I had pitched (01 SIGKILL, 16 hot-swap, 18 lock contention) turn out to verify *kernel/systemd contracts*, not project code: SIGKILL during `write(2)` is a kernel invariant; systemd-managed slot transitions are deploy-acceptance, not CI; `flock(2)` semantics are the same syscall whether across child JVMs in a test or across containers. So unit + integration (with `ProcessBuilder`-spawned child JVMs where cross-process matters, Testcontainers MinIO where real S3 semantics matter) covers everything that's *our code*. The soak alone validates the docker-compose wiring (env vars, mount points, port mappings, startup ordering, healthcheck targets, real MinIO, real systemd unit files) — and naturally exercises rotation and multi-node independence at real-env.
- **§12 ↔ test traceability via naming convention.** No spec-enumerated catalogue. Test class names reference the §12 entry they cover (`WsDisconnectIT` → §12.b, `Rest429StormUnitTest` → §12.l, etc.). If a new §12 entry has no matching test, `grep -r "§12.X" */src/test/` exposes the gap. Cheaper than maintaining a per-row table that drifts the moment someone renames a test.
- **Existing `01_collector_active_crash.sh` deleted.** Its recovery logic moves to a unit test in `collector/src/test/java` (recovery from manually-prepared dirty state); the restart-replay flow moves to an integration test with a child JVM via `ProcessBuilder`. The systemd-restart aspect of the original chaos test was deploy-acceptance, not CI value.
- **Soak triggers a rotation mid-run.** Picked over the "steady-state only" and "rotation + deploy" alternatives — single moving part is easy to diagnose if the soak fails, and rotation is the highest-value daily event in production anyway. Synthetic deploy stays out for now; can be added when hot-swap (Variant A) lands.

### Context / decisions
- **Audit found only 2 commits with TDD-debt, not the days of work I feared.** All other feat commits since the smoke test (`296a222`) shipped with at least one test file in the same commit. Pattern: TDD slipped specifically on the REST-poll feature and the forceOrder broadcast — both done in a "let me get this working" mode before validation. Everything else (multi-stream routing, dual-socket WS, sequence-ID gap detection, REST backfill) had tests attached.
- **Retrofit is characterization testing, not TDD.** The implementations already exist; the new tests lock in current behavior. If a test had revealed a bug, the fix would go in a separate commit. None did.
- **No HTTP timeout test for `RestPoller`.** The 30 s `REQUEST_TIMEOUT` is hard-coded and would make the test slow. Connection-refused exercises the same `writeError(...,null status)` envelope code path with a different `klass` string ("CONNECTION" vs "TIMEOUT"). The literal "TIMEOUT" string is the only thing not asserted; envelope shape is fully covered.
- **`RestPoller.queryString` order** is currently insertion-ordered via `LinkedHashMap`. The test pins that contract (`"symbol=BTCUSDT&limit=100"` in order). If Binance ever cares about param order or signed-request canonicalisation, this contract is greppable.
- **`PollerSchedulerTest` cadence tests run ~6.8 s combined** (1 s minimum cadence × 2.5 / 1.2 / 1.5 s sleeps). Acceptable for `mvn verify`. If they ever feel slow, refactor `PollerScheduler` to take a `Clock` and inject a fake clock — but that's a real refactor, not retrofit territory.
